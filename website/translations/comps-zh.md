# 可比公司分析

## ⚠️ 关键提示：数据源优先级（请先阅读）

**始终遵循以下数据源层级：**

1. **首先：检查MCP数据源** - 如果S&P Kensho MCP、FactSet MCP或Daloopa MCP可用，请专门使用这些数据源获取财务和交易信息
2. **不要使用网络搜索** - 如果上述MCP数据源可用
3. **仅在MCP不可用时：** 然后使用Bloomberg终端、SEC EDGAR文件或其他机构来源
4. **永远不要将网络搜索作为主要数据源** - 它缺乏机构级分析所需的准确性、审计追踪和可靠性

**这很重要：** MCP来源提供经过验证的机构级数据，并带有适当的引用。网络搜索结果可能过时、不准确或对金融分析不可靠。

## 概述
本技能教导Claude构建机构级的可比公司分析，结合运营指标、估值倍数和统计基准。输出是一个结构化的Excel/电子表格，通过对等比较支持明智的投资决策。

**参考资料和情境化：**

提供了一个可比公司分析示例在 `examples/comps_example.xlsx`。在使用本技能目录中的此或其他示例文件时，请智能地使用：

**可以使用示例来：**
- 理解结构层级（各部分如何流动）
- 掌握预期的严谨程度（统计深度、文档标准）
- 学习原则（清晰的标题、透明的公式、审计追踪）

**不要使用示例来：**
- 精确复制格式或指标
- 在不考虑上下文的情况下复制布局
- 无论受众如何都应用相同的视觉风格

**首先问自己：**
1. **"你有偏好的格式还是我调整模板风格？"**
2. **"受众是谁？"**（投资委员会、董事会演示、快速参考、详细备忘录）
3. **"关键问题是什么？"**（估值、增长分析、竞争定位、效率）
4. **"上下文是什么？"**（并购评估、投资决策、行业对标、业绩审查）

**根据具体情况调整：**
- **行业背景**：大型科技公司需要与新兴SaaS初创公司不同的指标
- **行业特定需求**：尽早添加相关指标（例如，针对科技公司的云ARR、企业客户、开发者生态系统）
- **公司熟悉度**：知名公司可能需要较少的背景，更多关注增量分析
- **决策类型**：并购需要与持续的投资组合监控不同的重点

**核心原则**：使用模板原则（清晰的结构、统计严谨性、透明的公式），但根据上下文调整执行。目标是机构质量的分析，而不是看起来像机构的模板。

用户提供的示例和明确的偏好始终覆盖默认设置。

## 核心理念

**"先建立正确的结构，然后让数据讲述故事。"**

从强制对重要事项进行战略思考的标题开始，输入干净的数据，构建透明的公式，统计数据自动出现。一个好的comps应该让没有构建它的人可以立即阅读。

## ⚠️ 关键提示：公式优于硬编码 + 逐步验证

**环境 - Office JS vs Python：**
- **在Excel中运行（Office插件/Office JS）：** 直接使用Office JS（`Excel.run(async (context) => {...})`）。通过 `range.formulas = [["=E7/C7"]]` 编写公式，而不是 `range.values`。不需要单独的重新计算步骤 - Excel原生处理计算。使用 `range.format.*` 进行颜色/字体设置。
- **生成独立的.xlsx文件：** 使用Python/openpyxl。编写 `cell.value = "=E7/C7"`（公式字符串）。
- 两种方式的原则相同 - 只需转换API调用。

**Office JS合并单元格陷阱：** 不要调用 `.merge()` 然后在合并范围上设置 `.values`（抛出 `InvalidArgument` - 范围仍然报告其合并前的维度）。而是仅在左上角单元格写入值，然后合并+格式化整个范围：
  ```js
  ws.getRange("A1").values = [["TECHNOLOGY — COMPARABLE COMPANY ANALYSIS"]];
  const hdr = ws.getRange("A1:H1");
  hdr.merge();
  hdr.format.fill.color = "#1F4E79";
  hdr.format.font.color = "#FFFFFF";
  hdr.format.font.bold = true;
  ```

**公式，而不是硬编码：**
- 每个派生值（利润率、倍数、统计）必须是一个引用输入单元格的Excel公式 - 永远不要粘贴预计算的数字
- 使用Python/openpyxl构建工作表时：编写 `cell.value = "=E7/C7"`（公式字符串），而不是 `cell.value = 0.687`（计算结果）
- 唯一允许的硬编码值是：原始输入数据（收入、EBITDA、股价等） - 每个值都有一个单元格注释说明其来源
- 原因：当输入更改时，模型必须自动更新。硬编码的利润率是一个等待发生的静默错误。

**逐步与用户验证：**
- 设置结构后 → 向用户显示标题布局，在填充数据之前
- 输入原始输入后 → 向用户显示输入块并确认来源/期间，然后再构建公式
- 构建运营指标公式后 → 向用户显示计算的利润率并在转向估值之前进行合理性检查
- 构建估值倍数后 → 向用户显示倍数并在添加统计之前确认它们看起来合理
- 不要端到端构建整个工作表然后展示 - 通过在每个部分确认来尽早捕获错误

## 第1部分：文档结构设置

### 标题块（第1-3行）
```
第1行：[分析标题] - COMPARABLE COMPANY ANALYSIS
第2行：[公司列表及代码] • [公司1 (TICK1)] • [公司2 (TICK2)] • [公司3 (TICK3)]
第3行：截至 [期间] | 所有数字以 [美元百万/十亿] 为单位，每股金额和比率除外
```

**这很重要：** 立即建立上下文。任何打开此文件的人都知道他们在看什么、何时创建的以及如何解释数字。

### 视觉约定标准（可选 - 用户偏好和上传的模板始终覆盖）

**重要：** 这些只是建议的默认值。始终优先考虑：
1. 用户明确的格式偏好
2. 任何上传的模板文件的格式
3. 公司/团队风格指南
4. 这些默认值（仅在没有提供其他指导时）

**建议的字体和排版：**
- **字体系列**：Times New Roman（专业、可读、行业标准）
- **字体大小**：数据单元格11pt，标题12pt
- **粗体文本**：章节标题、公司名称、统计标签

**默认颜色和阴影 - 专业蓝/灰色调（最小化更好）：**
- **保持最小化** - 仅使用蓝色和灰色。不要引入绿色、橙色、红色或多种强调色。一个干净的comps表总共使用3-4种颜色。
- **章节标题**（例如，"OPERATING STATISTICS & FINANCIAL METRICS"）：
  - 深蓝色背景（`#1F4E79` 或 `#17365D` 海军蓝）
  - 白色粗体文本
  - 所有列的整行阴影
- **列标题**（例如，"Company"、"Revenue"、"Margin"）：
  - 浅蓝色背景（`#D9E1F2` 或类似淡蓝色）
  - 黑色粗体文本
  - 居中对齐
- **数据行**：
  - 公司数据的白色背景
  - 公式的黑色文本；硬编码输入的蓝色文本
- **统计行**（最大值、第75百分位数等）：
  - 浅灰色背景（`#F2F2F2`）
  - 黑色文本，左对齐标签
- **这就是整个调色板**：深蓝 + 浅蓝 + 浅灰 + 白色。除非用户的模板另有说明，否则没有其他内容。

**建议的格式约定：**
- **小数精度**：
  - 百分比：1位小数（12.3%）
  - 倍数：1位小数（13.5x）
  - 美元金额：无小数，千位分隔符（69,632）
  - 利润率显示为百分比：1位小数（68.7%）
- **边框**：无边框（干净、最小化的外观）
- **对齐**：所有指标居中对齐，以获得干净、统一的外观
- **单元格尺寸**：所有列宽应均匀/相等，所有行高应一致（创建干净、专业的网格）

**注意：** 如果用户提供模板文件或指定不同的格式，请使用该格式。

## 第2部分：运营统计和财务指标

### 核心列（从此开始）
1. **Company** - 具有一致格式的名称
2. **Revenue** - 规模指标（可以是LTM、季度或年度，取决于上下文）
3. **Revenue Growth** - 同比百分比变化
4. **Gross Profit** - 收入减去销售成本
5. **Gross Margin** - 毛利润/收入（基本盈利能力）
6. **EBITDA** - 息税折旧摊销前利润
7. **EBITDA Margin** - EBITDA/收入（运营效率）

### 可选添加（根据行业/目的选择）
- **Quarterly vs LTM** - 如果季节性重要，包括两者
- **Free Cash Flow** - 对于资本密集型或SaaS业务
- **FCF Margin** - FCF/收入（现金生成效率）
- **Net Income** - 对于成熟、盈利的公司
- **Operating Income** - 对于D&A变化的公司
- **CapEx metrics** - 对于资产密集型行业
- **Rule of 40** - 专门用于SaaS（增长% + 利润率%）
- **FCF Conversion** - 用于盈利质量分析（高级）

### 公式示例（使用第7行作为示例）
```excel
// 核心比率 - 始终计算
Gross Margin (F7): =E7/C7
EBITDA Margin (H7): =G7/C7

// 可选比率 - 如果相关则包括
FCF Margin: =[FCF]/[Revenue]
Net Margin: =[Net Income]/[Revenue]
Rule of 40: =[Growth %]+[FCF Margin %]
```

**黄金法则**：每个比率应该是 [某物] / [收入] 或 [某物] / [此工作表中的某物]。保持简单。

### 统计块（在公司数据之后）

**关键提示：为所有可比指标（比率、利润率、增长率、倍数）添加统计公式。**

```
[留一个空行用于视觉分离]
- Maximum: =MAX(B7:B9)
- 75th Percentile: =QUARTILE(B7:B9,3)
- Median: =MEDIAN(B7:B9)
- 25th Percentile: =QUARTILE(B7:B9,1)
- Minimum: =MIN(B7:B9)
```

**需要统计的列（可比指标）：**
- Revenue Growth %、Gross Margin %、EBITDA Margin %、EPS
- EV/Revenue、EV/EBITDA、P/E、Dividend Yield %、Beta

**不需要统计的列（规模指标）：**
- Revenue、EBITDA、Net Income（绝对规模因公司规模而异）
- Market Cap、Enterprise Value（不同规模的公司之间不可比）

**注意：** 在公司数据和统计行之间添加一个空行以进行视觉分离。不要添加"SECTOR STATISTICS"或"VALUATION STATISTICS"标题行。

**四分位数很重要的原因：** 它们显示分布，而不仅仅是平均值。第75百分位倍数告诉你"优质"公司的交易价格。

## 第3部分：估值倍数和投资指标

### 核心估值列（从此开始）
1. **Company** - 与运营部分相同的顺序
2. **Market Cap** - 当前市场估值
3. **Enterprise Value** - 市值 ± 净债务/现金
4. **EV/Revenue** - 市场为每美元销售收入支付多少
5. **EV/EBITDA** - 市场为每美元收益支付多少
6. **P/E Ratio** - 价格相对于净收益

### 可选估值指标（根据上下文选择）
- **FCF Yield** - FCF/市值（用于关注现金的分析）
- **PEG Ratio** - P/E/增长率（用于成长型公司）
- **Price/Book** - 市场价值 vs 账面价值（用于资产密集型业务）
- **ROE/ROA** - 回报指标（用于盈利能力比较）
- **Revenue/EBITDA CAGR** - 历史增长率（用于趋势分析）
- **Asset Turnover** - 收入/资产（用于运营效率）
- **Debt/Equity** - 杠杆（用于资本结构分析）

**关键原则：** 包括对你的行业重要的3-5个核心倍数。不要仅仅因为可以就包括每个可能的指标。

### 公式示例
```excel
// 核心倍数 - 始终包括这些
EV/Revenue: =[Enterprise Value]/[LTM Revenue]
EV/EBITDA: =[Enterprise Value]/[LTM EBITDA]
P/E Ratio: =[Market Cap]/[Net Income]

// 可选倍数 - 如果有数据则包括
FCF Yield: =[LTM FCF]/[Market Cap]
PEG Ratio: =[P/E]/[Growth Rate %]
```

### 交叉引用规则
**关键提示：** 估值倍数必须引用运营指标部分。永远不要两次输入相同的原始数据。如果收入在C7中，那么EV/收入公式应该引用C7。

### 统计块
与运营部分相同的结构：每个指标的最大值、第75、中位数、第25、最小值。添加一个空行以进行视觉分离。不要添加"VALUATION STATISTICS"标题行。

## 第4部分：注释和方法论文档

### 必需组件

**数据源和质量：**
- 数据来自哪里？（S&P Kensho MCP、FactSet MCP、Daloopa MCP、Bloomberg、SEC文件）
- 涵盖什么期间？（2024年第四季度、审计数字）
- 如何验证？（与10-K/10-Q交叉检查）
- 注意：如果可用，优先使用MCP数据源（S&P Kensho、FactSet、Daloopa）以获得更好的准确性和可追溯性

**关键定义：**
- EBITDA计算方法（毛利润 + D&A，或营业利润 + D&A）
- 自由现金流公式（经营现金流 - CapEx）
- 解释特殊指标（Rule of 40、FCF Conversion）
- 期间定义（LTM、CAGR计算期间）

**估值方法论：**
- 企业价值如何计算？（市值 + 净债务）
- 使用了什么增长率？（历史CAGR、前瞻性估计）
- 做了哪些调整？（排除一次性项目、标准化利润率）

**分析框架：**
- 投资论点是什么？（云/SaaS效率）
- 哪些指标最重要？（现金生成、资本效率）
- 读者应该如何解释统计数据？（四分位数提供上下文）

## 第5部分：选择正确的指标（决策框架）

### 从"我在回答什么问题？"开始

**"哪家公司被低估了？"**
→ 关注：EV/Revenue、EV/EBITDA、P/E、市值
→ 跳过：运营细节、增长指标

**"哪家公司效率最高？"**
→ 关注：Gross Margin、EBITDA Margin、FCF Margin、Asset Turnover
→ 跳过：规模指标、绝对美元金额

**"哪家公司增长最快？"**
→ 关注：Revenue Growth %、EBITDA CAGR、用户/客户增长
→ 跳过：利润率指标、杠杆比率

**"哪一个是最好的现金生成器？"**
→ 关注：FCF、FCF Margin、FCF Conversion、CapEx intensity
→ 跳过：EBITDA、P/E比率

### 行业特定指标选择

**软件/SaaS：**
必须有：Revenue Growth、Gross Margin、Rule of 40
可选：ARR、Net Dollar Retention、CAC Payback
跳过：Asset Turnover、库存指标

**制造/工业：**
必须有：EBITDA Margin、Asset Turnover、CapEx/Revenue
可选：ROA、库存周转率、积压订单
跳过：Rule of 40、SaaS指标

**金融服务：**
必须有：ROE、ROA、效率比率、P/E
可选：净利息利润率、贷款损失准备金
跳过：Gross Margin、EBITDA（对银行没有意义）

**零售/电商：**
必须有：Revenue Growth、Gross Margin、库存周转率
可选：同店销售、客户获取成本
跳过：高额研发或CapEx指标

### "5-10法则"

**5个运营指标** - 收入、增长、2-3个利润率/效率指标
**5个估值指标** - 市值、EV、3个倍数
**= 总共10列** - 足以讲述故事，不会太多而失去线索

如果你有超过15个指标，你可能包括噪音。无情地编辑。

## 第6部分：最佳实践和质量检查

### 开始之前
1. **定义对等组** - 公司必须真正可比（相似的业务模式、规模、地理位置）
2. **选择正确的期间** - LTM平滑季节性；季度显示趋势
3. **提前标准化单位** - 数百万 vs 数十亿决策影响一切
4. **映射数据源** - 知道每个数字来自哪里

### 构建时
1. **首先输入所有原始数据** - 在编写公式之前完成蓝色文本
2. **为所有硬编码输入添加单元格注释** - 右键单击单元格 → 插入注释 → 记录来源 OR 假设

   **对于来源数据，准确引用来自哪里：**
   - 示例："Bloomberg Terminal - MSFT Equity DES，访问于2024-10-02"
   - 示例："2024年第四季度10-K文件，第42页，行项目'Total Revenue'"
   - 示例："FactSet共识估计截至2024-10-02"
   - **尽可能包括超链接**：右键单击单元格 → 链接 → 粘贴SEC文件、数据源或报告的URL

   **对于假设，解释推理：**
   - 示例："假设15% EBITDA利润率基于对等中位数，公司未披露"
   - 示例："估计企业价值为市值 + 5000万美元净债务（来自第三季度资产负债表，第四季度尚未可用）"
   - 示例："前瞻P/E基于街共识EPS 3.45美元（12位分析师估计的平均值）"

   **这很重要**：启用审计追踪、数据验证、假设透明度，并支持未来更新
3. **逐行构建公式** - 在继续之前测试每个计算
4. **对标题使用绝对引用** - $C$6锁定标题行
5. **一致地格式化** - 百分比为百分比，不是小数
6. **添加条件格式** - 自动突出异常值

### 合理性检查
- **利润率测试**：毛利率 > EBITDA利润率 > 净利润率（根据定义总是如此）
- **倍数合理性**：
  - EV/Revenue：通常0.5-20x（因行业而异）
  - EV/EBITDA：通常8-25x（各行业相当一致）
  - P/E：通常10-50x（取决于增长率）
- **增长-倍数相关性**：较高增长通常意味着较高倍数
- **规模-效率权衡**：较大公司通常有更好的利润率（规模效益）

### 避免的常见错误
❌ 混合市值和企业价值在公式中
❌ 对分子和分母使用不同的时间段（LTM vs 季度）
❌ 硬编码数字到公式中而不是单元格引用
❌ **硬编码输入没有单元格注释引用来源 OR 解释假设**
❌ 在可用时缺少到SEC文件或数据源的超链接
❌ 包括太多指标而没有明确目的
❌ 包括不可比的公司（不同的业务模式）
❌ 使用过时数据而不披露
❌ 错误地计算百分比的平均值（应该是中位数）

## 第6部分：高级功能

### 动态标题
对于显示计算的列，使用清晰的单位标签：
```
Revenue Growth (YoY) % | EBITDA Margin | FCF Margin | Rule of 40
```

### 四分位数分析好处
不仅仅是均值/中位数，四分位数显示：
- **第75百分位** = "优质"公司在这里交易
- **中位数** = 典型市场估值
- **第25百分位** = "折扣"区域

这有助于回答："我们的目标公司 vs 对手是交易丰富还是便宜？"

### 行业特定修改

**软件/SaaS：**
- 添加：ARR、Net Dollar Retention、CAC回收期
- 强调：Rule of 40、FCF利润率、毛利率 >70%

**医疗保健：**
- 添加：研发/收入、管道价值、监管状态
- 强调：EBITDA利润率、增长率、报销风险

**工业：**
- 添加：积压订单、订单簿趋势、地理组合
- 强调：ROIC、资产周转率、周期性调整

**消费：**
- 添加：同店销售、客户获取成本、品牌价值
- 强调：收入增长、毛利率、库存周转率

## 第7部分：工作流程和实用技巧

### 逐步流程
1. **设置结构**（30分钟）
   - 创建所有标题
   - 格式化单元格（输入为蓝色，公式为黑色）
   - 锁定单位和日期引用

2. **收集数据**（60-90分钟）
   - 从主要来源提取（S&P Kensho MCP、FactSet MCP、Daloopa MCP如果可用；否则Bloomberg、SEC）
   - 以蓝色输入所有原始数字
   - 在注释部分记录来源

3. **构建公式**（30分钟）
   - 从简单比率（利润率）开始
   - 进展到倍数（EV/Revenue）
   - 添加交叉检查（利润率有意义吗？）

4. **添加统计**（15分钟）
   - 为所有列复制公式结构
   - 验证范围正确（B7:B9，不是B7:B10）
   - 检查四分位数逻辑

5. **质量控制**（30分钟）
   - 运行合理性检查
   - 验证公式引用
   - 检查#DIV/0!或#REF!错误
   - 与已知基准比较

6. **文档记录**（15分钟）
   - 完成注释部分
   - 添加数据源
   - 定义方法论
   - 日期戳分析

### 专业技巧
- **保存模板**：构建一次，永远重复使用
- **颜色编码异常值**：为值 >2个标准差的单元格设置条件格式
- **链接到源文件**：超链接到Bloomberg屏幕截图或SEC文件
- **版本控制**：保存为"Comps_v1_2024-12-15"，日期清晰
- **协作审查**：让其他人检查你的公式

### Excel格式检查清单（可选 - 根据用户偏好调整）
- [ ] 字体设置为首选项（默认：Times New Roman，11pt数据，12pt标题）
- [ ] 章节标题根据用户模板格式化（默认：深蓝#17365D配白色粗体文本）
- [ ] 列标题根据用户模板格式化（默认：浅蓝/灰#D9E2F3配黑色粗体文本）
- [ ] 统计行根据用户模板格式化（默认：浅灰#F2F2F2）
- [ ] 没有应用边框（干净、最小的外观）
- [ ] **列宽设置为均匀/等宽**（创建干净、专业的外观）
- [ ] **行高设置为一致高度**（通常数据行为20-25pt）
- [ ] 数字格式化有正确的小数精度和千位分隔符

## 第10部分：红旗和警告信号

### 数据质量问题
🚩 不一致的时间段（混合季度和年度）
🚩 缺失数据没有解释
🚩 数据源之间显著差异（>10%方差）

### 估值红旗
🚩 负EBITDA公司用EBITDA倍数估值（改用收入倍数）
🚩 P/E比率 >100x 没有超增长故事
🚩 对行业没有意义的利润率

### 可比性问题
🚩 不同的财政年度结束（导致时间问题）
🚩 混合纯玩和集团
🚩 物质上不同的业务模式标记为"comps"

**有疑问时，排除公司。** 最好有3个完美的comps而不是6个有问题的。

## 第11部分：公式参考指南

### 基本Excel公式
```excel
// 统计函数
=AVERAGE(range)          // 简单平均值
=MEDIAN(range)           // 中间值
=QUARTILE(range, 1)      // 第25百分位
=QUARTILE(range, 3)      // 第75百分位
=MAX(range)              // 最大值
=MIN(range)              // 最小值
=STDEV.P(range)          // 标准差

// 财务计算
=B7/C7                   // 简单比率（利润率）
=SUM(B7:B9)/3            // 多家公司的平均值
=IF(B7>0, C7/B7, "N/A")  // 条件计算
=IFERROR(C7/D7, 0)       // 处理除零

// 跨工作表引用
='Sheet1'!B7             // 引用另一个工作表
=VLOOKUP(A7, Table1, 2)  // 从数据表查找
=INDEX(MATCH())          // 高级查找

// 格式化
=TEXT(B7, "0.0%")        // 格式化为百分比
=TEXT(C7, "#,##0")       // 千位分隔符
```

### 常见比率公式
```excel
Gross Margin = Gross Profit / Revenue
EBITDA Margin = EBITDA / Revenue
FCF Margin = Free Cash Flow / Revenue
FCF Conversion = FCF / Operating Cash Flow
ROE = Net Income / Shareholders' Equity
ROA = Net Income / Total Assets
Asset Turnover = Revenue / Total Assets
Debt/Equity = Total Debt / Shareholders' Equity
```

## 核心原则总结

1. **结构驱动洞察** - 正确的标题强制正确的思考
2. **少即是多** - 5-10个重要指标击败20个不重要指标
3. **为你的问题选择指标** - 估值分析 ≠ 效率分析
4. **统计显示模式** - 中位数/四分位数比平均值揭示更多
5. **透明度胜过复杂性** - 每个人都理解的简单公式
6. **可比性是王** - 最好排除而不是强制糟糕的comp
7. **记录你的选择** - 在注释部分解释哪些指标以及为什么

## 持续改进

完成comps分析后，问：
1. 统计数据是否显示了意外的洞察？
2. 是否有任何数据缺口限制了分析？
3. 利益相关者是否要求了你没有包括的指标？
4. 花了多长时间 vs 应该花多少时间？
5. 下次如何使这更有用？

最好的comps分析在每次迭代中演进。保存模板，从反馈中学习，并基于决策者实际使用的内容来优化结构。
